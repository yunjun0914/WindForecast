# 대회 규칙 정리 (rules)

작성일: 2026-07-09. 원문을 요약 정리한 것이며, 마지막 섹션은 우리 파이프라인에 대한 팀 해석이다.

## 1. Data Leakage 방지

기본 원칙:

- 각 예측값은 **해당 예측값의 예측기준시점 이전에 생성·공개·확정되어 실제로 활용 가능했던 정보만** 사용할 수 있다.
- 사용 가능 여부는 데이터가 가리키는 **대상 시각이 아니라, 생성·공개·확정된 시각** 기준으로 판단한다.
- 예측기준시점 이후에 관측·생성·공개·확정된 정보는 예측 대상 시각에 대한 정보라도 사용 불가.

예보 데이터:

- 예보의 사용 가능 여부는 예보 **생성·공개 시각** 기준.
- 예: 2025-01-13 10:00 발전량을 예측하는 행의 예측기준시점이 2025-01-12 14:00이면, 2025-01-13 00:00에 공개된 예보는 사용 불가.

실측 데이터:

- 예측기준시점 전에 이미 관측·공개·확정된 과거 자료만 사용 가능.
- **예측기준시점 이후의 실측값, 사후 보정자료, 재분석자료는 사용 불가.**

금지 정보:

- 평가 구간의 실제 발전량, 실측 운영 데이터, 비공개 운영 데이터, 정답에 준하는 정보.
- 정답(준하는 정보 포함)의 비공식 수집·활용은 즉시 실격.
- 파생 변수·통계값·보간값·집계값에도 예측기준시점 이후 정보가 포함되면 안 됨.

소명:

- 데이터와 전처리·학습·추론 과정의 규칙 준수를 코드 검증/2차 평가에서 설명할 수 있어야 한다.

## 2. 외부 데이터

사용 가능 조건 (모두 충족):

- 법적 제약 없음, 저작권·라이선스·개인정보·약관 위반 없음.
- **누구나 접근 가능한 공개 데이터.**
- Data Leakage 방지 규칙 준수.

제한:

- 특정 주체만 접근 가능한 비공개/내부 운영 데이터 불가.
- 정답을 포함하거나 유추 가능한 데이터 불가.

제출·재현 의무 (2차 평가 대상 시):

- 외부 데이터 파일, 출처, 수집 방법·시점, 사용 기간·변수, 라이선스, 전처리 코드 전부 제출.
- 제출된 코드와 자료만으로 재현 가능해야 함.
- 외부 데이터의 생성·공개·확정 시점이 각 예측값의 예측기준시점 이전임을 설명 가능해야 함.

## 3. 사전 학습모델

- **2026-07-05까지** 가중치가 공식 공개된 오픈소스 모델만 사용 가능.
- 사용·수정·배포·재배포·상업적 이용이 허용되는 라이선스여야 함 (비상업/연구 전용 제한 모델 불가).

## 4. API 기반 모델 금지

- OpenAI/Gemini/HF Inference/Together/OpenRouter 등 원격 API 추론 불가.
- 직접 관리하는 PC/서버에서 가중치를 직접 로드해 실행하는 방식만 허용.

## 5. 제출/평가

- 1일 최대 제출 5회.
- 모든 CSV는 UTF-8.
- **평가 데이터셋은 제출 파일(예측 결과) 생성을 위한 추론 목적으로만 사용 가능.**
- 최종 채점 파일은 제출 창에서 1개 선택.
- Private 랭킹은 최종 순위가 아니며 코드 검증·2차 평가 후 수상자 결정.

## 6. 우리 파이프라인에 대한 함의 (팀 해석)

1. **2025 pseudo-labeling(test-time adaptation)은 규칙 위반 소지가 높다.** "평가 데이터셋은 추론 목적으로만 사용"(5절) — 평가셋 weather를 학습 데이터로 넣는 test-time 학습은 추론 목적을 벗어난다. 기존 `g3_pseudo2022`는 **train 데이터셋**의 2022 weather를 쓴 것이므로 무관. → 2025 pseudo-labeling 아이디어는 보류/폐기.
2. **외부 데이터는 "train 기간 자료의 학습 활용" 형태가 안전하다.** 예: 기상청 AWS/ASOS 관측(공개, 2022~2024)을 teacher/보정 학습에 사용 — 공개·과거·재현 가능 요건 충족. 2025 예측 행 입력에는 각 행의 예측기준시점(전일 13:00경) 이전에 공개된 관측만 사용 가능하며, 공개 시점 소명 부담이 있다.
3. **재분석자료(ERA5 등)는 test 기간 입력으로는 사실상 불가**(사후 생성·지연 공개). train 기간 학습용으로는 공개 시점상 가능하나, 2차 평가 소명(공개·확정 시점) 준비 필요.
4. 사전학습 모델 쓸 경우 2026-07-05 이전 공개 + 상업적 이용 허용 라이선스만.
5. 예측기준시점은 데이터상 `data_available_kst_dtm`(전일 13:00경)으로 해석. 현재 파이프라인의 시퀀스/EMA/과거-시간 feature는 모두 그 이전 공개 예보만 쓰므로 적합.
